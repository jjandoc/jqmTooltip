About

jqmTooltip is a jQuery Mobile plugin for adding tooltip dialog boxes optimized for use on devices with smaller screens. Tooltip boxes are designed for smart positioning so they reveal, by default, in the mobile viewport to avoid scrolling or running off the page.

Demo

What's it look like?

Clicking on this button will activate the tooltip. By default, the tooltip will show centered underneath the element that activated it. If this would cause it to extend outside of the window horizontally, it will position itself with a minimum distance from the left or right sides of the window (a distance which can be defined in the options). If revealing below the element will cause it to be shown outside of the window, it will instead show above the element (this can also be adjusted in the options). While there is a "close" button, the overlay can be closed by clicking anywhere on the screen or resizing the window.

Another example

The button to the right is an example of the tooltip that will adjust horizontally so it does not show outside of the window. Notice that the arrow still adjusts position to be under its triggering element.

Usage

jqmTooltip requires jQuery and jQuery Mobile so you will need to include these libraries along with the jqmTooltip.css and jqmTooltip.js files. A typical <head> will include this:

<link rel="stylesheet" href="http://code.jquery.com/mobile/1.0b2/jquery.mobile-1.0b2.min.css" />
<link rel="stylesheet" href="css/jqmTooltip.css" />
<script type="text/javascript" src="http://code.jquery.com/jquery-1.6.2.min.js"></script>
<script type="text/javascript" src="http://code.jquery.com/mobile/1.0b2/jquery.mobile-1.0b2.min.js"></script>
<script type="text/javascript" src="js/jqmTooltip.js"></script>

Once you've got the plugin script in place, all you need to do is call the .jqmTooltip() method on your jQuery object. The .jqmTooltip method takes one optional object argument for plugin options. Typically you will target an <a> anchor element and the jqmTooltip will use the element's title attribute for its content (this can be changed in the options).

Since jQuery Mobile leverages a lot of AJAX and DOM manipulation, my javascript implementation tends to look something like this.

<script>
    $('div:jqmData(role="page")').live('pagecreate', function(e){		
        $(e.target).find('.tooltip').jqmTooltip();
    });
</script>

But if you're using a multi-page layout or not using JQM's AJAX page loads, you can get away with just using the $(document).ready event.

<script>
    $(document).ready(function(){
        $('.tooltip').jqmTooltip();
    });
</script>

As for the HTML, all you need is this (feel free to use whatever class/selector you wish).

<a class="tooltip" href="#" title="This is the text jqmTooltip uses by default">What's this?</a>

Options

jqmTooltip provides the following options and defaults

$('.tooltip').jqmTooltip({
    'arrow': true,
    'fadeSpeed': 200,
    'maxWidth': 280,
    'minSideMargins': 15,
    'position': 'autoUnder',
    'offset': 15,
    'text': 'title'
})
  • arrow: add an arrow to the tooltip. Boolean. Default = true
  • fadeSpeed: fade in/out animation speed in milliseconds. Use 0 for instant show/hide. Integer. Default = 200
  • maxWidth: maximum width in pixels of the tooltip overlay. Integer. Default = 280
  • minSideMargins: minimum horizontal distance in pixels the overlay will sit from the window edge. Integer. Default = 15
  • position: where the overlay will appear vertically in relation to the triggering element. Options are 'over' for always over, 'under' for always under, 'autoOver' for over unless it shows outside of the window, and 'autoUnder' for under unless it shows outside of the window. String. Default = 'autoUnder'
  • offset: vertical distance in pixels the tooltip (not including the arrow) will show above or below the triggering element. Integer. Default = 15
  • text: the text that will display in the tooltip. If it's the default, 'title', it will use the triggering element's title attribute. String or function. Default = 'title'
